# 从代码到落地部署指南

当前版本采用更简单的部署架构：前端 + 后端 + MySQL。

二维码图片不再上传到对象存储。后端根据二维码记录里的 `jumpPageUrl` 动态生成 PNG：

- 预览：`GET /api/qrcodes/{id}/image`
- 下载：`GET /api/qrcodes/{id}/image?download=true`

## 1. 服务

1. `frontend`：Vue 3 + Vite，构建后由 Nginx 提供静态文件和 `/api` 反向代理。
2. `backend`：Spring Boot，提供业务 API、扫码落地页数据和二维码图片生成接口。
3. `mysql`：保存小程序配置、二维码记录、扫码记录、活动配置。

## 2. 本地开发

```bash
docker compose up -d mysql
```

后端用 IDE 启动：

- Main class: `com.byd.qrcode.QrcodeChannelApplication`
- Profile: `dev`
- Port: `8080`

前端：

```bash
cd frontend
npm run dev -- --host 127.0.0.1 --port 5173 --strictPort
```

## 3. 单机生产部署

1. 复制环境变量：

```bash
cp .env.example .env
```

2. 修改 `.env` 中的域名、数据库密码和微信配置。

3. 执行部署：

```bash
bash deploy.sh
```

部署后访问：

- 前端：`http://localhost`
- 健康检查：`http://localhost/api/health`

## 4. 必要环境变量

```env
APP_BASE_URL=http://your-server-ip
APP_CORS_ALLOWED_ORIGINS=http://your-server-ip
VITE_API_BASE_URL=/api

DB_HOST=mysql
DB_PORT=3306
DB_NAME=qrcode_channel
DB_USERNAME=root
DB_PASSWORD=your_secure_db_password
MYSQL_DATABASE=qrcode_channel

WECHAT_MP_APPID=your_mp_appid
WECHAT_MP_SECRET=your_mp_secret
```
