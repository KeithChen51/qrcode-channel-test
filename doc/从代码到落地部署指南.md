# 从代码到落地部署指南

当前生产部署按“公司已有 Nginx + 后端 + MySQL”组织。

二维码图片不再依赖对象存储。后端根据当前公开访问地址动态生成 PNG，历史记录里即使曾写入 `localhost`，通过 Nginx 访问时也会按转发 Host 修正：
- 预览：`GET /api/qrcodes/{id}/image`
- 下载：`GET /api/qrcodes/{id}/image?download=true`

## 1. 服务边界

1. 公司 Nginx：托管 `frontend/dist` 静态文件，并把 `/api` 反向代理到后端。
2. `backend`：Spring Boot，提供业务 API、扫码落地页数据和二维码图片生成接口。
3. `mysql`：保存小程序配置、二维码记录、扫码记录和活动配置。

生产环境不再启动 `frontend` 容器。前端只需要构建成静态文件，交给公司 Nginx。

## 2. 本地开发

```bash
docker compose up -d mysql
```

后端用 IDE 启动：
- Main class: `com.byd.qrcode.QrcodeChannelApplication`
- Profile: `dev`
- Port: `8080`

前端：

```bash
cd frontend
npm run dev -- --host 127.0.0.1 --port 5173 --strictPort
```

## 3. 生产部署

1. 构建前端静态文件：

```bash
cd frontend
npm ci
npm run build
```

2. 把 `frontend/dist` 发布到公司 Nginx 站点目录，例如：

```bash
/opt/qrcode-channel/frontend/dist
```

3. 复制 Nginx 示例配置：

```bash
deploy/nginx/qrcode-channel.conf
```

确认反向代理保留公开域名和协议头：

```nginx
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
```

根据实际域名、证书和前端目录调整 `server_name`、`listen`、`root`。

4. 配置后端环境变量：

```bash
cp .env.example .env
```

5. 启动后端和 MySQL：

```bash
bash deploy.sh
```

默认情况下，后端只绑定到 `127.0.0.1:8080`，供公司 Nginx 在同一台服务器上反向代理。

## 4. 必要环境变量

```env
APP_BASE_URL=https://qrcode.company.com
APP_CORS_ALLOWED_ORIGINS=https://qrcode.company.com
BACKEND_BIND_HOST=127.0.0.1
BACKEND_PORT=8080

DB_HOST=mysql
DB_PORT=3306
DB_NAME=qrcode_channel
DB_USERNAME=root
DB_PASSWORD=your_secure_db_password
MYSQL_DATABASE=qrcode_channel

WECHAT_MP_APPID=your_mp_appid
WECHAT_MP_SECRET=your_mp_secret
```

`APP_BASE_URL` 建议显式配置为公网域名；如果暂时未配置，后端会使用 Nginx 转发的 `Host` / `X-Forwarded-Proto` 生成二维码图片地址和 H5 跳转地址。

如果使用公司已有 MySQL，把 `DB_HOST`、`DB_PORT`、`DB_NAME`、`DB_USERNAME`、`DB_PASSWORD` 改成公司数据库信息。`deploy.sh` 会跳过本项目自带 MySQL。

## 5. 验证

- 前端：`https://qrcode.company.com`
- 健康检查：`https://qrcode.company.com/api/health`
- 二维码预览：`https://qrcode.company.com/api/qrcodes/{id}/image`
- 二维码下载：`https://qrcode.company.com/api/qrcodes/{id}/image?download=true`

## 6. 回滚

保留上一版后端镜像或 JAR，以及上一版 `frontend/dist`。异常时回退静态文件目录和后端版本后重载 Nginx。
