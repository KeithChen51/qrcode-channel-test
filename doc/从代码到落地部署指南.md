# 从代码到落地部署指南

当前生产部署按“公司已有 Nginx + 后端 + MySQL”组织。

二维码图片不依赖 MinIO 或对象存储。后端按请求动态生成 PNG，用户下载图片时直接访问：

- 预览：`GET /api/qrcodes/{id}/image`
- 下载：`GET /api/qrcodes/{id}/image?download=true`

管理后台现在需要登录。首次启动时后端会根据环境变量创建一个管理员账号，首次登录后必须修改密码。

## 1. 服务边界

1. 公司 Nginx：托管 `frontend/dist` 静态文件，并把 `/api` 反向代理到后端。
2. `backend`：Spring Boot，提供管理 API、H5 落地页公开 API、扫码回传 API 和二维码图片生成接口。
3. `mysql`：保存管理员账号、小程序配置、二维码记录、扫码记录和活动配置。

生产环境不需要单独启动 `frontend` 容器。前端只构建为静态文件，交给公司 Nginx。

## 2. 本地开发

```bash
docker compose up -d mysql
```

后端用 IDE 启动：

- Main class: `com.byd.qrcode.QrcodeChannelApplication`
- Profile: `dev`
- Port: `8080`
- Environment:
  - `ADMIN_USERNAME=admin`
  - `ADMIN_INITIAL_PASSWORD=change_me_please_123`
  - `AUTH_TOKEN_SECRET=local_dev_secret_at_least_32_chars`
  - `AUTH_TOKEN_TTL_MINUTES=720`

前端：

```bash
cd frontend
npm run dev -- --host 127.0.0.1 --port 5173 --strictPort
```

## 3. 生产部署

1. 构建前端静态文件：

```bash
cd frontend
npm ci
npm run build
```

2. 把 `frontend/dist` 发布到公司 Nginx 站点目录，例如：

```bash
/opt/qrcode-channel/frontend/dist
```

3. 复制 Nginx 示例配置：

```bash
deploy/nginx/qrcode-channel.conf
```

确认反向代理保留公开域名和协议头：

```nginx
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
```

4. 配置后端环境变量：

```bash
cp .env.example .env
```

至少修改这些值：

```env
APP_BASE_URL=https://qrcode.company.com
APP_CORS_ALLOWED_ORIGINS=https://qrcode.company.com

ADMIN_USERNAME=admin
ADMIN_INITIAL_PASSWORD=replace_with_a_temporary_password
AUTH_TOKEN_SECRET=replace_with_a_random_secret_at_least_32_chars
AUTH_TOKEN_TTL_MINUTES=720

DB_HOST=mysql
DB_PORT=3306
DB_NAME=qrcode_channel
DB_USERNAME=root
DB_PASSWORD=your_secure_db_password
MYSQL_DATABASE=qrcode_channel

WECHAT_MP_APPID=your_mp_appid
WECHAT_MP_SECRET=your_mp_secret
```

`ADMIN_INITIAL_PASSWORD` 只在数据库里没有管理员账号时用于初始化。账号创建后不会覆盖已有密码，但建议仍然保留一个非空值，避免空库恢复时启动失败。

5. 启动后端和 MySQL：

```bash
bash deploy.sh
```

默认情况下，后端只绑定到 `127.0.0.1:8080`，供公司 Nginx 在同一台服务器上反向代理。

## 4. 已有数据库升级

如果不是全新数据库，需要先执行：

```bash
mysql -h <host> -P <port> -u <user> -p <db_name> < sql/migrations/20260528_create_admin_users.sql
```

然后重启后端。后端也会在启动时自动创建 `admin_users` 表；手工执行迁移主要用于运维变更留痕。

## 5. 验证

- 前端：`https://qrcode.company.com`
- 登录页：`https://qrcode.company.com/login`
- 后端健康检查：`https://qrcode.company.com/api/health`
- 二维码预览：`https://qrcode.company.com/api/qrcodes/{id}/image`
- 二维码下载：`https://qrcode.company.com/api/qrcodes/{id}/image?download=true`
- H5 跳转：`https://qrcode.company.com/jump?qid={id}`

注意：扫码二维码里的 H5 地址必须是 Nginx 对外地址，不应该是后端内网端口 `:8080`。

## 6. 回滚

保留上一版后端镜像或 JAR，以及上一版 `frontend/dist`。异常时回滚静态文件目录和后端版本后重载 Nginx。
